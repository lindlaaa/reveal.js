Git

What is it?
  • an unpleasant or contemptible person.
    • "Git: a version control system for tracking changes in computer files and coordinating work on those files among multiple people."
    ↓

    a filesystem

    ↓
    Git is a file system, like windows explorer, build for linux in the early 2000's by Linus Torvalds. Not only was it part of the Linux filesystem, but it has been used to develop Linux since.

    A timeline

    ↓
    It not only stores your current files, but records every change made, who made it, as well as a message from the person who made it.

    A backup

    ↓
    At any point you can revert to an older "commit" point, and your actual files on your disk will change to reflect the state of the file system when the "commit" was made.

    A "shared" filesystem

    It allows you to work

    with

    other developers and not

    against!

    This is where our new tools come into play.

    Working with others is easy. You can work off of each others commits. To do this we can use the command line, but there are tools that can make life easier and take us away from the command line.

    bitbucket

    What is it?
    "Distributed version control system that makes it easy for you to collaborate with your team. The only collaborative Git solution that massively scales."
    ↓
    Bitbucket is the basically the server that all of our sourcecode is stored on.

    Our repository of code

    ↓
    You cannot do any actual development on bitbucket, but you can manage all of the commits and sets of commits called branches.

    Source code management tool

    ↓
    This is where we do our code reviews, merge my changes with yours, "checkout" each others code to make sure all of the unit tests pass, and where we eventually merge all good code into production.

    What is it really?

    It is really just a linux machine, with a fancy GUI attached to it. Underneath are basic Git commands.

    Sourcetree

    What is it?
    "SourceTree simplifies how you interact with your Git repositories so you can focus on coding."
    ↓
    Sourcetree is really just a GUI to, once again, use Git on your local computer.

    Whats the difference?

    Everything you can do on BitBucket you can also do on SourceTree, however, others cannot see it!

    ↓
    This doesnt just mean you can do things without others knowing, but implies that you can test, modify, improve, even break things, without bothering other developers.

    But what if I like my changes?

    Good news, you can just tell BitBucket about your changes with a "push", and now others will be able to see them!

    SourceTree makes this very easy, and even updates BitBucket by default. It not only abstracts that task away, but also lets you know when other "branches" you are working on are "out-of-date".

    Why these tools?

    ↓
    There are other tools out there that can accomplish some of the same things, and there are some out there that even do parts of it better, however, there are none that integrate so well with each other. BitBucket and SourceTree are made to be used in a professional development environment together.
